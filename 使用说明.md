# “下一组”iPhone 像素训练节拍器

这是一个可以添加到 iPhone 主屏幕的个人 Web App。训练计划、当前进度和历史记录都保存在手机本地，不需要登录。

## 已实现

- 可编辑动作、组数、次数、重量和每个动作的休息时间
- 可建立多个训练计划，并在首页选择今天训练哪一个
- 经典像素冒险界面：营地、关卡、史莱姆伙伴和营火休息计时
- 休息期间显示循环举杠铃的像素小人、跳动营火和火星动画
- 完成本组后，调用名为“组间提醒”的 iPhone 快捷指令启动系统计时器
- 不再使用会跳进 Safari 的网页回调；可在快捷指令末尾尝试用“打开 App”返回“下一组”
- 切到游戏、其他 App 或锁屏后，系统计时器仍会继续并响铃
- 自动保存训练计划和未完成进度
- 添加到 iPhone 主屏幕后可离线打开

## 发布到 GitHub Pages

把本文件夹中的全部文件上传到 GitHub 仓库根目录，并在仓库的 `Settings → Pages` 中选择：

- Source：`Deploy from a branch`
- Branch：`main`
- Folder：`/(root)`

## 添加到 iPhone 主屏幕

1. 使用 Safari 打开 GitHub Pages 网址。
2. 点击“共享”。
3. 选择“添加到主屏幕”。
4. 开启“作为网页 App 打开”，然后点击“添加”。

## 配置“组间提醒”快捷指令

1. 打开 iPhone 自带的“快捷指令”App，新建快捷指令。
2. 名称必须设置为 **组间提醒**。
3. 添加“从快捷指令输入获取数字”。
4. 添加“计时”，时间使用上一步得到的数字，单位为“秒钟”。
5. 在最后添加“打开 App”。如果 App 列表里能找到“下一组”，请选择它；找不到就删除这一步。
6. 保存快捷指令。
7. 回到“下一组”，打开“计时魔法”，点击“测试 10 秒系统计时器”。
8. 测试成功后，点击“我已设置完成”。

## 关于返回主屏幕 App

HTTPS 回调会被 iPhone 交给 Safari，而不是已经安装到主屏幕的网页 App，因此新版已经移除网页回调。主屏幕网页 App 无法像原生 App 一样注册专属 URL scheme。如果快捷指令的“打开 App”列表能找到“下一组”，可以用它直接返回；如果找不到，只能从 App 切换器手动返回，或将项目进一步制作成原生 iPhone App。

## 多计划使用

1. 在首页点击右上角“＋”新建训练计划。
2. 编辑计划名称、动作、组数、次数、重量和休息秒数，然后点击“完成”。
3. 重复操作建立推、拉、腿等多个计划。
4. 每次训练前，在首页上方点击一个训练存档卡片，再点击“开始冒险”。
5. 训练开始后当前计划会锁定，结束训练后才可以切换，避免训练进度错乱。

## 更新后仍显示旧界面

先在 Safari 打开网页并刷新一次，再关闭并重新打开主屏幕上的“下一组”。如果仍是旧界面，删除主屏幕旧图标，然后重新“添加到主屏幕”。训练数据存在 Safari 网站数据中，清除网站数据会删除本地训练记录。

## 文件结构

- `index.html`：App 入口
- `app.js`：训练与计时逻辑
- `styles-pixel.css`：像素冒险界面
- `manifest.webmanifest`：主屏幕 Web App 配置
- `service-worker.js`：离线缓存
- `icon-512.png` / `icon.svg`：像素主屏幕图标
